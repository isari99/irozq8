1) ربط Twitch Chat
    •    استخدام مكتبة tmi.js للاتصال بشات Twitch.
    •    الاتصال بقناة معينة (يتم تحديد اسم القناة).
    •    الاستماع لرسائل الشات.
    •    التقاط الإجابات عندما يكتب المستخدم:
    •    1 أو 2 أو 3 أو 4 فقط
    •    ربط كل إجابة باسم المستخدم (Twitch username).
    •    تخزين الإجابات لكل سؤال.

⸻

2) نظام الأسئلة
    •    إنشاء بنك أسئلة (Questions Bank).
    •    كل سؤال يحتوي على:
    •    نص السؤال
    •    4 خيارات
    •    الإجابة الصحيحة
    •    عرض سؤال واحد في كل مرة.

⸻

3) منع تكرار الأسئلة
    •    استخدام نظام Shuffle للأسئلة.
    •    عرض الأسئلة بشكل عشوائي بدون تكرار.
    •    عدم إعادة السؤال نفسه إلا بعد انتهاء جميع الأسئلة.
    •    عند انتهاء جميع الأسئلة يتم إعادة خلطها من جديد.

⸻

4) نظام اللعب
    •    يوجد متغير يتحكم في:
    •    السؤال الحالي
    •    حالة الجولة
    •    عند بدء اللعبة:
    •    يتم اختيار أول سؤال
    •    عند الضغط على “السؤال التالي”:
    •    الانتقال للسؤال التالي بدون تكرار

⸻

5) لوحة التحكم (Host Controls)
    •    زر:
    •    Start Game (بدء اللعبة)
    •    Start Question (بدء السؤال)
    •    Next Question (السؤال التالي)
    •    التحكم يكون فقط من قبل الـ Host.
6) احتساب الإجابات
    •    عند استقبال إجابات من الشات:
    •    مقارنة إجابة المستخدم مع الإجابة الصحيحة
    •    تسجيل الإجابات الصحيحة
    •    تخزين نقاط لكل مستخدم:
    •    +1 لكل إجابة صحيحة

⸻

7) لوحة المتصدرين (Leaderboard)
    •    عرض قائمة اللاعبين داخل اللعبة:
    •    الاسم
    •    عدد النقاط
    •    تحديث مباشر بعد كل سؤال.
    •    ترتيب اللاعبين تنازلياً حسب النقاط.

⸻

8) واجهة المستخدم
    •    عرض السؤال في منتصف الشاشة.
    •    عرض 4 خيارات تحت السؤال.
    •    إزالة الشات من الواجهة.
    •    إضافة مكان مخصص للشعار (Logo).
    •    تصميم بسيط ومنظم.

⸻

9) تدفق اللعبة
    
تشغيل السيرفر
الاتصال بشات Twitch
بدء اللعبة من لوحة التحكم
استقبال إجابات الشات
عرض النتائج
تحديث المتصدرين
الانتقال للسؤال التالي

⸻

10) ملاحظات مهمة
    •    يجب أن يعمل التطبيق بالكامل داخل Replit.
    •    استخدام Node.js + Express (إن لزم).
    •    استخدام WebSocket أو API لتحديث الواجهة بشكل لحظي.
    •    التأكد من أن الشات فقط هو مصدر الإجابات.
    •    لا يوجد نظام “join” نهائياً.
    •    أي مستخدم في الشات يمكنه المشاركة مباشرة بكتابة رقم الإجابة.
