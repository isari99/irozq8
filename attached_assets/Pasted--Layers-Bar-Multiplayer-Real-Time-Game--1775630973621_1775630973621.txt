إنشاء لعبة Layers Bar (Multiplayer Real-Time Game)



📌 
ملاحظة قبل البدء:


    •    سأقوم بإرسال: 
    ◦    صور الشخصيات التي سيتم استخدامها داخل اللعبة
    ◦    صورة/تصميم البار (Environment)
    •
    •    المطلوب منك: 
    ◦    استخدام هذه الصور داخل اللعبة
    ◦    بناء مشاهد سينمائية (Cinematic Intro)
    ◦    دمج الشخصيات داخل البيئة بشكل واقعي
    •





🌐 
نظام الدخول (رابط واحد - Lobby):


    •    يتم إنشاء رابط دخول واحد للعبة
    •    عند فتح الرابط: 
    ◦    يدخل اللاعب إلى Lobby
    ◦    يظهر زر Join / Start
    ◦    اللاعب يضغط انضمام أو يتم تسجيله تلقائيًا
    •
    •    عند الانضمام: 
    ◦    يتم سحب: 
    ▪    اسم اللاعب
    ▪    صورته (Avatar / Profile Image)
    ◦
    ◦    يتم عرض اللاعب داخل قائمة اللاعبين
    •
    •    عند وصول 4 لاعبين: 
    ◦    يتم قفل الغرفة (Lobby Lock)
    ◦    يبدأ العد التنازلي أو يبدأ Host اللعبة
    •





🎮 
نظام Host (المضيف):


    •    يوجد لاعب (Host) لديه زر: 
    ◦    Start Game
    •
    •    عند الضغط: 
    ◦    تبدأ اللعبة لجميع اللاعبين
    •





🎭 
اختيار الشخصيات:


    •    كل لاعب يختار شخصية من الصور التي سيتم إرسالها
    •    يتم تمثيل اللاعب داخل اللعبة كشخصية 2D/3D حسب التصميم
    •    يتم توزيع اللاعبين حول طاولة البار





🎬 
المشهد السينمائي (Intro):


    •    عند بداية اللعبة: 
    ◦    يتم تشغيل مشهد سينمائي: 
    ▪    الشخصيات تدخل إلى البار
    ▪    تمشي وتجلس حول الطاولة
    ◦
    ◦    الكاميرا تتحرك بشكل سينمائي (Cinematic Camera)
    •
    •    بعد ذلك يتم الانتقال إلى gameplay





🍸 
تصميم البيئة:


    •    استخدام صورة البار التي سيتم إرسالها كخلفية رئيسية
    •    عرض: 
    ◦    طاولة بار في المنتصف
    ◦    4 مقاعد حول الطاولة
    ◦    اللاعبين يجلسون عليها
    •





🃏 
نظام الأوراق (Cards System):


    •    يتم توزيع أوراق عشوائية لكل لاعب
    •    الأوراق تحتوي على: 
    ◦    King / Ace / Queen / Numbers … إلخ
    •
    •    كل لاعب يرى أوراقه فقط
� 
نظام التحكم (Keyboard Controls):


    •    A / D: 
    ◦    التنقل بين الأوراق
    •
    •    Space: 
    ◦    اختيار الورقة
    •
    •    E: 
    ◦    رمي/لعب الورقة
    •





🎯 
طريقة اللعب:


    •    يتم تحديد نوع الورق المطلوب لكل دور (مثلاً: King)
    •    اللاعب يجب أن: 
    ◦    يلعب أوراق مطابقة أو يكذب ويضع أوراق غير مطابقة
    •
    •    يمكن اختيار عدد أوراق (1 إلى 3 مثلاً) ووضعها دفعة واحدة





👀 
نظام الكذب والكشف:


    •    اللاعب الذي يأتي بعدك مباشرة (على يمينك فقط) هو الوحيد الذي يمكنه: 
    ◦    كشفك (Call Bluff)
    •
    •    عند الشك: 
    ◦    يتم كشف الأوراق
    •



النتائج:


    •    إذا كان اللاعب كاذب: 
    ◦    يتم تنفيذ Animation إطلاق على نفسه
    ◦    يخسر HP / طلقات
    •
    •    إذا كان صادق: 
    ◦    اللاعب الذي شك يطلق على نفسه
    •





🔫 
نظام الطلقات (HP System):


    •    كل لاعب لديه عدد محدود من المحاولات (مثلاً 7)
    •    عند خسارة الطلقات: 
    ◦    يتم إخراجه من اللعبة
    •





🔄 
تبديل الأدوار:


    •    اللعب يكون بنظام الأدوار (Turn-based)
    •    كل لاعب يأخذ دوره بالتتابع
    •    يتم تحديث الدور لكل لاعب بشكل واضح





🏁 
نهاية اللعبة:


    •    آخر لاعب يبقى حي هو الفائز
    •    يتم عرض: 
    ◦    اسمه
    ◦    شخصيته
    ◦    احتفال بالفوز
    •
� 
المتطلبات البصرية:


    •    دمج صور الشخصيات المرسلة داخل اللعبة
    •    استخدام صورة البار كخلفية رئيسية
    •    تنفيذ مشاهد دخول سينمائية (Cinematic Intro)
    •    جعل الجلسة داخل البار واقعية ومناسبة للأجواء
    •    ترتيب اللاعبين حول الطاولة بشكل منظم





⚙️ 
متطلبات تقنية:


    •    Multiplayer Real-Time (WebSocket أو ما يعادله)
    •    Room System (Lobby)
    •    State synchronization بين اللاعبين
    •    Turn-based logic
    •    Input handling لكل لاعب
    •    دعم الانضمام عبر الرابط مباشرة





🎯 
ملاحظات مهمة:


    •    يجب أن تكون التجربة: 
    ◦    سلسة
    ◦    تفاعلية
    ◦    بدون تعقيد في الدخول
    •
    •    كل لاعب يعتمد على: 
    ◦    الماوس + الكيبورد
    •
    •    يجب عدم خلط اللاعبين أو بياناتهم
    •    الحفاظ على استقرار الجلسة حتى نهاية اللعبة
